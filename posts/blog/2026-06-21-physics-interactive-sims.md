把國高中物理常見的抽象概念，做成一個個**可以即時拖、即時看的互動模擬**，集合成一個平台。力學、熱學、波動、光學、電磁、近代物理六大領域，目前累積 30 多個模擬，每一個都能拉滑桿調參數，畫面、圖表、公式跟著一起變。

![物理互動模擬平台首頁：六大領域分類](/assets/blog/physics-home.png)

這篇不只介紹有哪些模擬，而是把背後那條「**拖滑桿 → 重算物理 → 重繪畫面 → 更新數據**」的技術鏈路完整拆給你看：動畫迴圈怎麼跑、參數怎麼綁、波和拋體各自用什麼數值方法算。全部純前端、不需要任何後端，開網頁就能用。

## 為什麼想做這個

物理難，常常難在**看不見**。課本上一張靜態圖，學生得自己在腦中把它動起來——把波往前推、把球拋出去、把分子撞來撞去。這個「在腦中跑模擬」的能力，恰恰是初學者最缺的。

模擬的價值，是把「**如果我把這個調大會怎樣？**」這個問題，從一段需要想像的推理，變成**一秒就能驗證的動作**。學生拖一下頻率滑桿，拍頻包絡立刻浮現；改一下拋射角，軌跡同步重畫。想像的負擔被還給了畫面，剩下的腦力可以拿去想「為什麼」。

這個平台屬於「AI 協作互動作品」（專案名 `ai-physics-demos`）——用 AI 輔助快速把一個個物理情境做成可玩的網頁，再由我這個現場老師來校正物理正確性、調整教學節奏。對生活科技課來說，它同時也是一個很好的「資料視覺化／模擬」跨域教具範例：示範**怎麼把一個數學模型，變成即時互動的視覺化工具**。

## 一張圖看懂背後的資料流

![互動模擬的即時資料流](/assets/blog/diagram-physics.svg)

每一個模擬，本質上都是同一個閉環在跑：

1. **使用者輸入**：拖動 `<input type="range">` 滑桿，改掉某個物理參數（頻率、角度、溫度…）。
2. **物理模型**：用數值方法算出「這一幀」整個系統的狀態。
3. **重繪畫面**：透過 `requestAnimationFrame` 把狀態畫到 Canvas 上，每秒約 60 次。
4. **即時讀數**：把關鍵物理量（頻率差、能量、座標…）同步更新到旁邊的公式與數字。

關鍵在於這是個**閉環**：動畫迴圈每一幀都會回到第二步重算，所以畫面是「活的」；而滑桿一動，只是把模型的輸入參數換掉，下一幀就自動反映出來。沒有後端、沒有資料庫，全部在瀏覽器裡完成。

## 技術一：requestAnimationFrame 動畫迴圈

整個平台的心臟，是一個標準的 `requestAnimationFrame`（rAF）迴圈。它的好處是**跟著螢幕更新率走**（通常 60 Hz），瀏覽器分頁切到背景時會自動暫停，比 `setInterval` 省電也更穩。

骨架長這樣：

```javascript
// 概念示意，非實際原始碼
let last = performance.now();

function frame(now) {
  const dt = (now - last) / 1000;   // 換算成「秒」，這一幀經過多久
  last = now;

  update(dt);   // 用 dt 把物理狀態往前推進
  draw();       // 把目前狀態畫到 canvas

  requestAnimationFrame(frame);     // 排下一幀，形成迴圈
}
requestAnimationFrame(frame);
```

這裡有個容易被忽略、但**物理正確性很關鍵**的細節：`update(dt)` 一定要吃「真實經過的時間 `dt`」，而不是假設每幀都剛好 1/60 秒。因為不同裝置、不同負載下，幀與幀的間隔會浮動；用實際 `dt` 來積分，模擬速度才不會在手機上變慢、在高刷新率螢幕上變快。

## 技術二：滑桿綁定物理參數

互動的入口，是把 HTML 的 range 滑桿綁到物理參數上。使用者拖動時觸發 `input` 事件，即時重算、即時重繪：

```javascript
// 概念示意，非實際原始碼
const slider = document.getElementById('freq1');

slider.addEventListener('input', () => {
  state.f1 = parseFloat(slider.value);   // 滑桿值 → 物理參數
  readoutEl.textContent =
    `f₁ = ${state.f1.toFixed(1)} Hz，頻率差 |f₁−f₂| = ${Math.abs(state.f1 - state.f2).toFixed(1)} Hz`;
  // 不必手動 draw()——rAF 迴圈下一幀就會用新的 state.f1 重畫
});
```

用 `input` 事件（而不是 `change`）的原因是：`input` 在「拖動過程中持續觸發」，`change` 只在「放開滑桿時觸發一次」。要做到「拖到哪、畫面就跟到哪」的即時感，必須用 `input`。

至於重繪，只要動畫迴圈一直在跑，事件處理函式裡其實**不需要自己呼叫 `draw()`**——改掉 `state` 之後，下一幀 rAF 自然會用新值重畫。這讓事件處理保持輕薄，也避免同一幀內重複繪製。

## 技術三：兩種數值方法——解析式取樣 vs. 逐幀積分

不同物理情境，用的數值方法不一樣。平台裡大致分成兩類，這也是寫模擬時最該想清楚的選擇。

### A. 波——直接用解析式取樣

波動類（行進波、駐波、雙波疊加）有現成的數學式，不需要積分，直接對 `x` 取樣畫出來就好。以精選範例「**干涉與摩爾紋：同一種疊加**」為例，它把兩個頻率 f₁、f₂ 的波相加：

```
y(x) = sin(2π·f₁·x) + sin(2π·f₂·x)
```

兩個相近頻率相加，會浮現一個低頻的「**拍頻包絡（beat）**」，其包絡週期為 `1 / |f₁ − f₂|`——頻率差越小，包絡越長、節點越疏。畫法就是沿著畫布寬度對 `x` 取樣：

```javascript
// 概念示意，非實際原始碼
function drawSuperposition(ctx, f1, f2, W, H) {
  ctx.beginPath();
  for (let px = 0; px <= W; px++) {
    const x = px / W;                          // 正規化到 [0,1]
    const y = Math.sin(2 * Math.PI * f1 * x)
            + Math.sin(2 * Math.PI * f2 * x);  // 兩波疊加
    const py = H / 2 - (y / 2) * (H / 2) * 0.9; // 振幅 ±2 映射到畫布
    px === 0 ? ctx.moveTo(px, py) : ctx.lineTo(px, py);
  }
  ctx.stroke();
}
```

![干涉與摩爾紋：雙波疊加浮現拍頻包絡](/assets/blog/physics-superposition.png)

這個模擬的教學威力在於**把好幾個看似無關的現象收斂成同一件事**：雙狹縫干涉、繞射條紋、兩張網點疊出來的摩爾紋（Moiré）——本質上都是「兩個週期性圖案疊加 → 產生差頻」。學生在這裡建立的直覺，可以一路遷移到光學單元。

### B. 拋體 / 力學——逐幀數值積分

力學類（拋體、碰撞、雲霄飛車、簡諧運動）牽涉「狀態隨時間演化」，就得逐幀積分。以拋體為例，用 **半隱式 Euler（symplectic Euler）**：每幀先更新速度、再用新速度更新位置。

```javascript
// 概念示意，非實際原始碼
function update(dt) {
  const g = 9.8;          // 重力加速度 (m/s²)
  ball.vy += -g * dt;     // 先更新速度
  ball.x  += ball.vx * dt;
  ball.y  += ball.vy * dt; // 再用更新後的速度更新位置
}
```

為什麼選半隱式 Euler 而不是「顯式 Euler」（先用舊速度更新位置）？因為半隱式 Euler 在週期性、有能量守恆的系統（單擺、彈簧、軌道）裡**長期穩定、能量不會無限漂移**，這對「能量守恆雲霄飛車」「行星軌道沙盒」這種要跑很久的模擬特別重要——順序對調一行，視覺上幾分鐘後就會看出軌道發散或衰減。對純拋體這種短程運動，等加速度的解析公式也可以，但統一用半隱式 Euler 能讓加上空氣阻力、碰撞等擴充時不必改架構。

## 涵蓋的六大領域

平台目前的模擬橫跨六大領域，每一個都對應課本裡一個常見的抽象難點：

| 領域 | 代表模擬（節選） |
|------|------------------|
| 力學 | 一維等加速度運動、拋體運動實驗室、能量守恆雲霄飛車、碰撞實驗台、牛頓第二定律（推箱子）、斜面與力的分解、等速圓周運動、角動量守恆（溜冰者效應）、簡諧運動三表徵、行星軌道沙盒、力的合成、槓桿與滑輪、浮力與密度、帕斯卡原理／液壓機 |
| 熱學 | 分子運動箱（PV=nRT）、比熱與加熱曲線、理想氣體 PV 圖、熱傳導與熱平衡、熱機／卡諾循環 |
| 波動 | 行進波（橫波）、駐波弦（共振與諧波）、空氣柱共鳴管、都卜勒效應與音爆、聲波合成器（拍音） |
| 光學 | 折射與反射、透鏡成像、雙狹縫干涉、繞射 等 |
| 電磁 | 電場與電力線、電路、電磁感應 等 |
| 近代物理 | 光電效應、原子能階、波粒二象 等 |

以單一模擬展開，可調參數與對應物理量的關係，就是學生「動手探索」的旋鈕。以雙波疊加為例：

| 可調參數 | 對應物理量 | 即時看到的變化 |
|----------|------------|----------------|
| 頻率 f₁ 滑桿 | 第一個波的頻率 | 波形變密／變疏 |
| 頻率 f₂ 滑桿 | 第二個波的頻率 | 同上，並改變與 f₁ 的差 |
| —（自動計算） | 頻率差 \|f₁−f₂\| | 拍頻包絡的長度（差越小越長） |
| —（自動計算） | 視窗內拍頻節點數 | 包絡上振幅趨近零的位置數 |

## 課堂怎麼用（對應 108 課綱）

這個專案雖然內容偏物理科，但放在**生活科技「科技與工程」**的脈絡下，它是一個很好的跨域教具與成果展示工具：

- **設計與製作 / 資料視覺化**：它本身就是「把一個數學模型做成即時互動視覺化工具」的完整範例。可以帶學生看「滑桿 → 模型 → 畫面」這條資料流，理解一個互動程式是怎麼運作的，呼應科技領域對「運算思維」與「資訊科技應用」的強調。
- **科技的本質**：示範抽象的物理規律如何被「程式化、參數化、可操作化」——這正是工程把理論落地的過程。
- **講解時當示範**：邊講邊拉參數，比放靜態投影片更有感；學生親眼看到「調大這個、那個就跟著變」，因果關係一目了然。
- **讓學生自己探索**：給一個開放問題（例如「怎樣讓拍頻最明顯？」），讓他們自己調到答案，再回來解釋為什麼——把課堂從「聽講」翻轉成「探究」。
- **零安裝、可分組**：純前端、開網頁就能用，**手機平板也能玩**，很適合分組讓每組一台裝置自己操作。

核心理念是一句話：**即時互動模擬，幫學生建立物理直覺**。把「看不見」變成「拖得到」。
