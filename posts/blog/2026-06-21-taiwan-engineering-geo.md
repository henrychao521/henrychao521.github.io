把真實的台灣工程，變成一場「看圖猜位置」的地圖遊戲——畫面給你一張衛星空照圖，可能是一座橋、一座水庫、一間半導體廠或一個車站，你要在地圖上點出它在台灣的哪裡，點得越近、分數越高。猜完揭曉正解，順帶認識這個結構背後的設計決策，再附上維基百科延伸閱讀。

![台灣工程地景探索首頁：台灣地圖、200 選、4 主題、151 維基條目](/assets/blog/geo-landing.png)

這篇把整個遊戲從玩法到實作拆給你看：互動地圖怎麼做、分數怎麼算、200 個景點的題庫長什麼樣，以及它在生活科技與跨域課堂上能怎麼用。全程純前端，一個瀏覽器就跑得動。

## 為什麼想做這個

我一直覺得，「工程設計」不該只在教室裡用課本談。台灣其實到處都是精彩的工程決策——為什麼這座橋要做成斜張橋、為什麼這棟高樓之間要架跨棟通道、為什麼這座水庫蓋在這個河段——只是我們很少把「它在哪裡」和「它為什麼這樣設計」放在一起看。

傳統上教工程地景，不外乎放幾張照片、講幾個案例，學生看過就忘，因為**那些工程跟他腦中的台灣地圖是脫節的**。如果能讓學生先用衛星圖**辨識位置、再回頭認識結構與設計動機**，這兩件事就被縫回同一張地圖了。地理猜謎這種玩法（看圖、點位置、比距離）天生就適合做這件事：它強迫你先觀察地形與工程結構的線索，再做出空間判斷——而觀察與判斷，正是工程素養的起點。

## 一題的遊戲迴圈長這樣

![台灣工程地景的單題遊戲迴圈：出題 → 猜測 → 算距離 → 換算分數 → 揭曉正解](/assets/blog/diagram-geo.svg)

整個遊戲的核心其實只有一個迴圈，每一題都重複它：

1. **出題**：左側顯示一張工程地景的衛星空照圖（Esri/Maxar 等來源）或實景街景，並給你文字提示——所在地區、設施類型、以及「觀察重點」。影像可以按「拉遠／拉近」切換層級，從遠處看地形脈絡、拉近看工程結構。
2. **玩家猜測**：右側是一張可互動的台灣地圖，你點一下放下猜測標記，按「確認猜測」送出。
3. **算距離**：系統用 Haversine 公式，算出你的猜測點與正解之間的**球面距離**（公里）。
4. **換算分數**：把距離丟進一個遞減函式，距離越近、分數越高，每題最高 1000 分。
5. **揭曉正解**：地圖上畫出「猜測點 → 正解」的連線、標出距離與本題得分，並展開這個地景的結構與設計重點，附維基百科連結讓你延伸學習。

下面三個技術段落，分別拆解這條迴圈裡最關鍵的三塊：地圖互動、距離計分、題庫與保存。

## 技術一：用 Leaflet 做可點擊的台灣地圖

地圖是這個遊戲的主舞台。我用的是 **[Leaflet](https://leafletjs.com/)**——一個輕量的開源網頁地圖函式庫，搭配 OpenStreetMap／CARTO 的免費底圖；衛星影像層則疊上 **Esri World Imagery**。純前端、不需要任何後端伺服器，這對要部署在 GitHub Pages 上的教學專案很重要。

互動的邏輯其實很單純：監聽地圖的 `click` 事件，在玩家點擊的經緯度放下一個標記，記住這個座標當作「猜測點」。

```javascript
// 概念示意（非實際原始碼）：點地圖放下猜測標記
let guessMarker = null;
let guessLatLng = null;

map.on('click', (e) => {
  guessLatLng = e.latlng;                 // { lat, lng }
  if (guessMarker) map.removeLayer(guessMarker);
  guessMarker = L.marker(guessLatLng).addTo(map);
  enableConfirmButton();                  // 有了猜測點才允許「確認」
});
```

揭曉答案時，再把「猜測點」和「正解點」用一條 `polyline` 連起來，並把地圖視野自動縮放到剛好框住這兩點，讓玩家一眼看出自己差了多遠：

```javascript
// 概念示意（非實際原始碼）：揭曉時畫出猜測 → 正解的連線
function revealAnswer(guess, answer) {
  L.marker(answer).addTo(map);                          // 正解位置
  const line = L.polyline([guess, answer], {
    color: '#ef4444', weight: 3, dashArray: '6 6'
  }).addTo(map);
  map.fitBounds(line.getBounds(), { padding: [40, 40] }); // 框住兩點
}
```

這種「先讓玩家承諾一個答案、再揭曉並視覺化差距」的設計，比起直接給標準答案，學習效果好很多——因為玩家在連線出現的那一刻，會自己去反思「我為什麼會猜錯這麼遠／這麼準」。

## 技術二：Haversine 距離與距離換算分數

計分是這個遊戲的數學核心，分成兩步：先算距離、再把距離換成分數。

**第一步：球面距離。** 地球是一個近似球體，兩個經緯度之間的最短距離是沿著球面的大圓弧長，不能直接用平面的畢氏定理。標準作法是 **Haversine 公式**。給定兩點的緯度 φ₁、φ₂ 與經度差 Δλ（都先轉成弧度），令 Δφ = φ₂ − φ₁：

```javascript
// Haversine：兩個經緯度 → 公里（概念示意，公式為標準正確式）
function haversineKm(lat1, lon1, lat2, lon2) {
  const R = 6371;                          // 地球平均半徑（公里）
  const toRad = (d) => d * Math.PI / 180;

  const dLat = toRad(lat2 - lat1);
  const dLon = toRad(lon2 - lon1);
  const a =
    Math.sin(dLat / 2) ** 2 +
    Math.cos(toRad(lat1)) * Math.cos(toRad(lat2)) *
    Math.sin(dLon / 2) ** 2;

  const c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
  return R * c;                            // 球面距離（公里）
}
```

寫成數學式就是：

```
a = sin²(Δφ/2) + cos φ₁ · cos φ₂ · sin²(Δλ/2)
c = 2 · atan2(√a, √(1−a))
d = R · c        （R ≈ 6371 km）
```

用 `atan2` 而不是 `asin`，是因為它在距離很大、`a` 接近 1 時數值更穩定。對台灣這種尺度（南北約四百公里）其實怎麼算都很準，但養成用標準式的習慣是好事。

**第二步：距離換算分數。** 有了距離 `d`，要把它變成「越近越高、有上限」的分數。一個常見作法是指數遞減：距離為 0 時拿滿分，距離越大、分數掉得越快但永遠不會變負數。

```javascript
// 距離 → 分數（示意公式，非唯一解；D0 控制分數衰減的快慢）
function distanceToScore(distKm) {
  const MAX = 1000;          // 每題滿分
  const D0 = 50;            // 特徵距離（公里）：越小，懲罰越嚴格
  const score = MAX * Math.exp(-distKm / D0);
  return Math.round(score);
}
```

這裡的 `D0` 是一個「特徵距離」，決定分數掉多快：以這個示意式為例，差 0 公里拿滿分、差 D0 公里大約剩三分之一強的分數。實務上要依台灣的尺度去調，讓「猜在同一個縣市」和「猜到隔壁島」之間有明顯的分數落差。用幾個級距來感受一下這條曲線：

| 與正解距離 | 大致語意 | 示意得分（D0=50） |
|------------|----------|------------------|
| 0 km | 正中紅心 | 1000 |
| 5 km | 同一個區 | 約 905 |
| 25 km | 同一個縣市 | 約 607 |
| 100 km | 猜錯區域 | 約 135 |
| 300 km | 整個猜反 | 約 2 |

> 提醒：上表的分數是用示意公式 `1000·exp(−d/D0)` 算出來的、用來說明「遞減」的概念，不是專案保證的實際數值；真正上線時的曲線會依題庫分布與教學手感微調。重點在於那條曲線的形狀：近距離給得慷慨、遠距離迅速歸零，玩家才會有動機去「再看仔細一點」。

## 技術三：結構化題庫與 localStorage 進度保存

200 個工程地景，每一個都是題庫裡的一筆**結構化資料**。把每題該有的欄位固定下來，出題、提示、揭曉、計分就都能用同一套程式跑：

```javascript
// 概念示意（非實際原始碼）：一筆題目的資料結構
const spot = {
  id: 'taichung-city-hall',
  name: '臺中市政府新市政大樓',
  lat: 24.1610, lng: 120.6402,           // 正解座標（用於 Haversine）
  region: '臺中市',                        // 地區提示
  facility: '行政／辦公建築',               // 設施類型提示
  theme: 'architecture',                  // 所屬主題模組（四選一）
  observe: '橫向巨型窗框、跨棟通道；高層間連通空橋以滑動支承（sliding bearing）因應地震差異沉陷',
  images: ['far.jpg', 'mid.jpg', 'near.jpg'], // 三種影像層級（由遠到近）
  wiki: 'https://zh.wikipedia.org/wiki/...'    // 維基延伸閱讀
};
```

注意 `observe` 這個欄位——它不只是答案說明，而是**這題真正要教的東西**。以臺中市政府新市政大樓為例，觀察重點是它的橫向巨型窗框與跨棟通道，以及**高層建築之間用連通空橋、以滑動支承（sliding bearing）來因應地震時的差異沉陷**。玩家猜完位置、揭曉的同時，也順手學到一個具體的工程設計決策。整套 200 題裡，這樣的「設計決策小知識」累積起來就很可觀，並有 151 條維基條目可以再往下挖。

進度保存則靠瀏覽器內建的 **`localStorage`**——它讓玩家不必登入、不必後端資料庫，就能把「檔案」（進度與成就）留在自己的瀏覽器裡：

```javascript
// 概念示意（非實際原始碼）：把進度存進瀏覽器
function saveProfile(profile) {
  localStorage.setItem('teg_profile', JSON.stringify(profile));
}
function loadProfile() {
  const raw = localStorage.getItem('teg_profile');
  return raw ? JSON.parse(raw) : { score: 0, played: [], badges: [] };
}
```

代價是進度綁在「這台裝置的這個瀏覽器」上，換裝置就看不到——但對課堂單機使用來說，這個取捨非常划算：零部署成本、零個資疑慮，純靜態檔丟上 GitHub Pages 就能用。

## 五種玩法，對應不同課堂節奏

同一條遊戲迴圈，包裝成五種模式，對應不同的使用情境——從不用登入的快速試玩，到可長期累積的完整挑戰：

| 模式 | 題數 | 涵蓋範圍 | 是否保存 | 特點 |
|------|------|----------|----------|------|
| 試玩 | 20 題 | 全部 200 景點 | 否 | 不需登入、隨開隨玩 |
| 精選地景暖身 | 每局 5 題 | 10 個經典地景 | 是 | 五分鐘炒熱氣氛 |
| 完整挑戰 | 每局 10 題 | 200 課程景點、四主題可選 | 是 | 主課程、可分組競賽 |
| 深度模式 | 揭曉後追問 2 題 | — | 是 | 答對每題 +300 分，逼你看更細 |
| Mapillary 即時街景 | — | 街景涵蓋處 | 是 | 需自備免費 token，看實景而非空照 |

我自己最喜歡「深度模式」的設計：它在玩家揭曉位置後，**再針對這個地景追問兩題**（答對每題加 300 分）。這逼著學生不能只滿足於「猜對在哪」，還得回頭把影像和提示看得更仔細——這正是從「地理辨識」走向「工程觀察」的關鍵一步。而 Mapillary 街景模式則讓有興趣深入的學生切換成第一人稱實景，從地面視角觀察工程結構。

## 課堂怎麼用（對應 108 課綱）

這個遊戲在生活科技課的定位，是把抽象的「工程設計流程」掛回**具體的台灣地景**上：

- **工程與設計**：每一題的「觀察重點」都是一個真實的設計決策——耐震（如滑動支承）、跨域整合（橋梁、廠房選址）、結構選型。學生從「成果地景」反推「設計動機」，練的正是工程思維裡最珍貴的那一段：理解約束條件如何形塑設計。
- **跨域整合**：一條遊戲迴圈同時串起**地理**（空間定位、地形判讀）、**工程**（結構與機構）、**資訊**（地圖互動、距離演算法、計分模型）。它本身就是「跨域」這個概念最好的示範案例。
- **台灣在地連結**：200 個景點全都在台灣，學生玩著玩著，腦中那張「台灣工程地圖」就一格一格被填起來——這是任何國外教材都給不了的在地連結。

實際操作上，建議用「精選地景暖身」開場（五題、五分鐘），把氣氛帶起來；主課程用「完整挑戰」並指定一個主題，搭配學習單做分組競賽；對學有餘力的學生，再用「深度模式」或 Mapillary 街景往下挖。
