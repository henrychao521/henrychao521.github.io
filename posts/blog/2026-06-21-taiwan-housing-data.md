把政府公開的房市資料，變成一個你可以「自己問問題」的儀表板——下拉選單切到任一個縣市，上面四個關鍵指標的數字、下面橫跨十年的價量軌跡圖就即時重畫一次。它的價值不在於告訴你「房子該不該買」，而在於示範一件事：**當你手上有真實資料，論述就不必憑感覺**。

![全台房市實證數據系統 v5.2 儀表板（高雄市為例）](/assets/blog/housing-dashboard.png)

這篇把整條資料流從頭拆給你看：原始開放資料長什麼樣、為什麼要先清洗成結構化 JSON、雙軸圖怎麼配、那幾個衍生指標是怎麼算出來的，以及它在「科技與社會」「資料素養」的課堂上能怎麼帶。

## 為什麼想做這個

生活科技課談「資料蒐集與分析」，最怕的就是停在抽象層。課本講「要會判讀統計圖表」「要有實證精神」，但學生實際接觸到的，往往是別人整理好、甚至替你下好結論的圖卡。

房市是個好題材，因為它**每個學生家裡都有感、新聞天天在講、而且資料是公開的**。內政部的不動產實價登錄、住宅價格指數都是開放資料，任何人都能下載。問題是：原始檔案是逐筆交易、欄位雜亂的 CSV，一般人根本看不出趨勢。

所以這個專案的核心，不是去「預測房價」，而是把**這些散落的官方原始資料，整理成一個學生可以直接對著問問題的介面**——切換縣市、比較地區、看同一條趨勢換個起點會不會講出不同的故事。這正是資料素養（data literacy）的核心：不是相信數字，而是會追問「這個數字是跟誰比、用什麼當基準」。

整個系統標示著 `GROUNDING DATA: 2026.03 OFFICIAL RELEASE`——刻意把「資料基準時間」放在最顯眼的地方，就是要提醒看的人：每一個數字都對應到一份可追溯的官方資料，而不是憑空生成。這就是「實證（grounding）」這個詞在這裡的意思。

## 整體資料流一張圖看懂

![全台房市數據系統的資料流](/assets/blog/diagram-housing.svg)

整條鏈路只有一個輸入（一批官方開放資料）和一個輸出（一個純前端的互動網頁），中間經過「清洗 → 視覺化 → 衍生指標」三個階段，最後靠一個下拉選單把 34 個縣市串起來即時切換。下面逐段拆解。

## Step 1 — 把雜亂的開放資料清洗成結構化 JSON

開放資料最大的門檻，從來不是「拿不到」，而是「拿到了也很難用」。實價登錄是逐筆交易紀錄、住宅價格指數是另一份按季發布的表，欄位命名、單位、缺漏狀況都不一致。

第一步是**把不同來源對齊到同一個時間軸（季度），並依縣市分組**，整理成一個前端好用的結構。我的做法是把它 reshape 成「每個縣市一組季度序列」的形狀：

```javascript
// 概念示意（非實際原始碼）：把原始開放資料正規化成「每縣市一組季度序列」
// 目標結構：county -> { quarters, priceIndex, volume }
const dataset = {
  "臺北市": {
    quarters:   ["2015Q1", "2015Q2", /* …連續到 */ "2025Q4"],  // 44 個季度
    priceIndex: [/* 對應住宅價格指數的形狀 */],                  // 折線用
    volume:     [/* 對應年度成交量分配 */],                      // 柱狀用
  },
  "高雄市": { quarters: [/* … */], priceIndex: [/* … */], volume: [/* … */] },
  // …全台 34 個縣市資料節點
};
```

這個結構的重點在於：**所有縣市共用同一條 `quarters` 時間軸**。一旦對齊，後面要切換縣市、要計算季變動、要疊圖比較，全部都只是換一組同樣長度的陣列，不必再回頭處理日期解析。清洗階段把髒活做完，視覺化階段才會乾淨。

> 清洗時最容易踩的坑：缺漏季度。如果某縣市某一季沒有資料，直接讓陣列「跳號」，折線和柱狀的 X 軸就會對不齊。寧可保留位置、填入 `null`，讓 Chart.js 自己處理斷點，也不要默默把它擠掉。

## Step 2 — Chart.js 的「折線＋柱狀」雙軸混合圖

這是整個儀表板視覺上的主角：一張橫跨 2015–2025、共 44 季的「實證價量連續軌跡」圖。**藍色折線是價格指數（套用內政部住宅價格指數的形狀）、灰色柱狀是對齊年度的總成交量分配**，兩者共用同一條時間軸，但各自有自己的 Y 軸刻度。

為什麼要雙軸？因為「價格（萬/坪）」和「成交量（棟）」是兩種完全不同量級的東西，硬畫在同一個刻度上，其中一個會被壓扁到看不出變化。Chart.js 的混合圖（mixed chart）剛好能處理這件事——同一張圖裡放一個 `bar` dataset 和一個 `line` dataset，再給它們各自指向不同的 `yAxisID`：

```javascript
// 概念示意（非實際原始碼）：Chart.js 雙軸混合圖設定
const config = {
  type: 'bar',                              // 基底是柱狀
  data: {
    labels: county.quarters,               // 共用的季度時間軸
    datasets: [
      {
        type: 'line',                      // 疊一條折線上去
        label: '實證單價指數',
        data: county.priceIndex,
        yAxisID: 'yPrice',                 // 走左軸（價格）
        tension: 0.3,                      // 稍微平滑
      },
      {
        label: '成交量（棟）',
        data: county.volume,
        yAxisID: 'yVolume',                // 走右軸（量）
      },
    ],
  },
  options: {
    scales: {
      yPrice:  { position: 'left',  title: { text: '實證單價（萬/坪）' } },
      yVolume: { position: 'right', title: { text: '成交棟數' },
                 grid: { drawOnChartArea: false } },   // 右軸不重複畫格線
    },
  },
};
```

實作上有兩個細節值得提：一是 `drawOnChartArea: false`，讓右軸不要再畫一層格線，否則兩套格線疊在一起會很亂；二是把折線的 dataset 放進來時要指定 `type: 'line'`，因為頂層 `type` 是 `bar`，不指定的話它會被當成柱狀。

## Step 3 — 衍生指標：四個數字都從同一組序列算出來

儀表板上方那四張卡片——**實證單價、結轉棟數、Q4 急凍幅度（QoQ）、五年度價格成長率**——不是另外存的資料，而是從同一組季度序列「即時算」出來的。這一點很重要：資料只有一份，所有指標都是它的函數，所以切換縣市時不會出現「圖和數字對不上」的狀況。

| 指標 | 定義 | 反映什麼 |
|------|------|----------|
| 實證單價（萬/坪） | 取序列最近一期的單價 | 當前價格水準 |
| 結轉棟數（棟） | 對應期別的實際成交棟數 | 真實流動性（有沒有人在買） |
| Q4 急凍幅度（QoQ） | (本季 − 上季) ÷ 上季 | 季對季的短期變動，看市場是熱是冷 |
| 五年度價格成長率 | (末期 ÷ 基準期) − 1，基準 = 2020 Q1 | 中期累積漲跌幅 |

其中兩個百分比指標的算法，本身就是很好的資料識讀教材，因為它們都牽涉到「跟誰比」這個問題：

```javascript
// 概念示意（非實際原始碼）：兩個關鍵衍生指標的算法

// QoQ 季變動：本季相對「上一季」的變化
function qoq(series) {
  const last = series.at(-1);
  const prev = series.at(-2);
  return (last - prev) / prev;       // 例如 -0.375 → 顯示 -37.5%
}

// 五年度成長率：末期相對「固定基準季（2020 Q1）」的累積變化
function fiveYearGrowth(series, quarters, baseQ = '2020Q1') {
  const baseIdx = quarters.indexOf(baseQ);
  const base = series[baseIdx];
  const last = series.at(-1);
  return last / base - 1;            // 末期 ÷ 基準 − 1
}
```

把這兩段並排看，資料素養的重點就浮出來了：**同樣叫「成長率」，QoQ 是跟「上一季」比、五年成長率是跟「一個固定基準」比，得到的故事完全不同。** 一個強調短期急凍、一個強調長期累積——這正是課堂上要讓學生意識到的：看到一個百分比，先問「分母是誰」。

## Step 4 — 一個下拉選單，串起 34 個縣市的即時重繪

最後把所有縣市接起來的，是上方那個「觀測區域」下拉選單。它的邏輯其實很單純：**監聽 `change` 事件 → 換成對應縣市的資料集 → 重算四個指標 → 呼叫 `chart.update()` 重繪**，全程不重新整理頁面、不打任何後端 API。

```javascript
// 概念示意（非實際原始碼）：切換縣市時的即時重繪
countySelect.addEventListener('change', (e) => {
  const county = dataset[e.target.value];   // 取出該縣市那組序列

  // 1) 換圖的資料
  chart.data.labels = county.quarters;
  chart.data.datasets[0].data = county.priceIndex;  // 折線
  chart.data.datasets[1].data = county.volume;      // 柱狀
  chart.update();                                   // 即時重繪

  // 2) 重算上方四張卡的衍生指標
  renderMetrics({
    price:  county.priceIndex.at(-1),
    volume: county.volume.at(-1),
    qoq:    qoq(county.priceIndex),
    growth: fiveYearGrowth(county.priceIndex, county.quarters),
  });
});
```

因為資料在 Step 1 就已經全部清洗、對齊成同樣的結構，這裡的切換才能這麼乾淨——它只是「換一組同樣長度的陣列」而已。**前期把資料處理乾淨，後期互動就會輕鬆**，這個取捨在任何資料專案都成立。

下面這張表把整條資料處理流程濃縮成四步，剛好對應課綱「資料蒐集與分析」的完整循環：

| 階段 | 在做什麼 | 產物 |
|------|----------|------|
| ① 蒐集 | 取得內政部實價登錄、住宅價格指數等開放資料 | 原始 CSV / 逐筆交易 |
| ② 清洗 | 正規化、對齊季度時間軸、依縣市分組 | 結構化 JSON（每縣市一組序列） |
| ③ 視覺化 | Chart.js 雙軸混合圖（折線＋柱狀） | 44 季價量連續軌跡圖 |
| ④ 衍生 | 由序列即時計算 QoQ、五年成長率等 | 四張指標卡 + 下拉即時重繪 |

## 課堂怎麼用（對應 108 課綱）

這個專案最適合用在**「資料蒐集與分析」以及「科技與社會」**這兩塊，重點完全不在房市本身，而在資料素養：

- **資料蒐集與開放資料應用**：帶學生去內政部開放資料平台看一眼原始檔案，再對照這個儀表板，體會「原始資料」到「可讀資訊」中間隔了多少清洗工。這比空談「政府有開放資料」具體得多。
- **統計圖表判讀**：給一個縣市，讓學生先自己描述「這十年的價量走勢」，再去對照新聞報導，看自己讀到的跟媒體說的有沒有出入。
- **實證思維（用真實資料說話）**：刻意拿 QoQ 和五年成長率這兩個指標當對照——同一個縣市，可以同時是「這一季急凍」又是「五年大漲」。讓學生理解，數字沒有騙人，是**「跟誰比」決定了故事**。
- **資料視覺化倫理**：反過來當作「怎麼呈現才不會誤導」的範例。同樣一條折線，截不同的起點、用不同的 Y 軸縮放，給人的印象天差地遠——這是「科技與社會」很好的討論切入點。

需要提醒的是：這是一個**資料識讀的教學工具，不是投資建議**。課堂上要守住的界線是「我們在練怎麼讀資料、怎麼問問題」，而不是「老師覺得哪裡會漲」。實際交易請以官方登錄為準。
