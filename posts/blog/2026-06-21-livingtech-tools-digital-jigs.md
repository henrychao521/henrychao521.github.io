生活科技課最珍貴、也最讓人捏一把冷汗的時刻，往往是學生**第一次握住線鋸機、按下手電鑽、把工件壓上鑽床**的那幾秒。實作教室時間有限、機台數量有限，老師很難一個一個盯著看。所以我做了一整套**數位教具**：讓「第一次」的緊張，先發生在瀏覽器裡。

![數位教具的工具櫃：一整排可點進去練習的機具卡片](/assets/blog/tools-list.png)

口號很簡單——**「動手前，先在數位環境練熟」**。學生在進實作教室之前，先在電腦或平板上把每件機具的部件、安全要點、操作步驟走過一遍，建立正確的概念、安全意識，甚至一點點肌肉記憶。這篇把整套教具的設計理念、五段式闖關架構，以及背後純前端的實作邏輯，完整拆給同行老師看。

## 為什麼想做這個

生活科技的機具教學有三個結構性的痛點，靠多排幾節課很難根治：

1. **安全風險前置不足**：學生對機具的危險性沒有具體想像，第一次操作就要同時顧「怎麼用」和「怎麼不受傷」，認知負荷太高。
2. **機台是稀缺資源**：一個班三十幾個人、機台可能只有幾台，多數時間在排隊等待，真正動手的時間被稀釋。
3. **快慢差異無處安放**：手快的學生等到無聊、手慢的學生被趕著走，差異化幾乎做不到。

數位教具不是要取代實作——**實作永遠是核心**——而是把「可以先在螢幕上練的部分」抽出來，挪到實作之前。學生先在數位環境把安全與概念練熟，進到實作教室時，老師就能把寶貴的現場時間，留給真正需要手把手的細節。這也呼應 108 課綱生活科技領域裡「機具安全操作」與「設計與製作」的核心精神：**安全前置，降低實作風險。**

## 一件機具 = 五段式闖關

整套教具最核心的設計，是每一件機具都拆成同一套**「五段式闖關」**流程。學生照順序闖，而且**第二關（安全）必須滿分通過，才會解鎖後面的實作模擬**——這是整個設計的靈魂。

![每件機具的五段式闖關流程](/assets/blog/diagram-livingtech-tools.svg)

橫向五段從左到右，難度與「動手程度」逐段遞增：先認識它、再確認你懂安全、然後看它怎麼動、接著自己模擬操作、最後認識它會怎麼壞。下面這張表把每一段「在教什麼、檢核什麼」攤開：

| 關卡 | 名稱 | 在教什麼 | 怎麼檢核 |
|------|------|----------|----------|
| ① | 認識部件 | 機具各部位的名稱、功能與相對位置 | 互動探索，點過熱點累積「已認識 X / N」 |
| ② | 安全闖關 | 操作前的安全意識與危險辨識 | 安全選擇題，**需滿分才解鎖後續** |
| ③ | 步驟動畫 | 正確的操作步驟與順序 | 逐步示範，建立流程心智模型 |
| ④ | 操作模擬器 | 在參數化情境下「試做」一次 | 切割 / 鑽孔互動，依材料與工具給回饋 |
| ⑤ | 故障 / 事故圖鑑 | 異常情境：為什麼會壞、怎麼預防 | 情境圖鑑，連結到安全與保養觀念 |

把「安全」卡在第二關、而且要滿分，是刻意的設計：**沒有在數位環境通過安全關，就進不到後面的操作模擬。** 它把「安全」從一句口頭叮嚀，變成一道學生必須親手跨過的門檻。

## 認識部件：三種視角 + SVG 互動熱點

第一關「認識部件」是學生對機具的第一印象，所以做得最豐富。每件機具提供**三種視角**讓學生對照：可點熱點的**平面圖解**、可旋轉縮放的 **3D 視角**、以及**實物照片**。三種視角各有所長——平面圖好標註、3D 好建立空間感、實照最貼近他們真正會看到的樣子。

![線鋸機「認識部件」：圖中橘色數字是可點的部件熱點](/assets/blog/tools-scrollsaw.png)

以線鋸機（scrollsaw）為例，模組一「認識線鋸機」放了 **8 個部件熱點**。學生點圖中的橘色數字，右側面板就會跳出該部件的**中英文名稱、功能說明，外加一條「冷知識」**——例如：彈一下鋸條，若發出清亮的「叮」聲，代表張力調得剛好。畫面右上角持續顯示「已認識 X / 8 個部位」，把「探索」變成一種有進度感的小遊戲，學生會想把 8 個全部點開。

技術上，這一關用 **SVG 互動熱點**實作：在機具圖上疊一層透明的 SVG，每個部件對應一個可點擊的圖形，點擊事件去顯示對應說明、並把它記進「已認識」集合。概念示意如下：

```html
<!-- 概念示意，非實際原始碼：機具圖上疊一層可點的 SVG 熱點 -->
<svg class="hotspots" viewBox="0 0 800 600">
  <circle class="spot" data-part="blade"    cx="410" cy="250" r="16"/>
  <circle class="spot" data-part="tension"  cx="560" cy="120" r="16"/>
  <!-- …其餘部件… -->
</svg>
```

```javascript
// 概念示意，非實際原始碼：點熱點 → 顯示說明 + 累計「已認識」
const seen = new Set();
const parts = {
  blade:   { zh: "鋸條", en: "Blade", desc: "上下往復切割工件…",
             fact: "彈鋸條發出清亮『叮』聲，代表張力適中。" },
  tension: { zh: "張力鈕", en: "Tension Knob", desc: "調整鋸條鬆緊…",
             fact: "張力不足鋸條容易跑線、甚至斷裂。" },
  // …其餘部件…
};

document.querySelectorAll('.spot').forEach(spot => {
  spot.addEventListener('click', () => {
    const key = spot.dataset.part;
    showPanel(parts[key]);          // 右側面板顯示名稱 / 說明 / 冷知識
    seen.add(key);
    updateCounter(seen.size, Object.keys(parts).length); // 「已認識 X / 8」
  });
});
```

用 SVG 而不是在圖上排一堆絕對定位的 `<div>`，好處是**熱點會跟著圖一起縮放**：不管學生用桌機大螢幕還是平板，熱點永遠對得準部件位置，這對 RWD 很重要。

## 安全闖關：滿分才解鎖的 gating 邏輯

第二關是整套設計的關鍵。學生要先答完一組**安全選擇題**，而且**全對（滿分）**，系統才把後面的「操作模擬器」解鎖。沒過？那就回去重看部件、重答安全題，直到通過為止。

這個「闖關 gating」邏輯本身不複雜，但它是把「安全」從建議變成**前置條件**的關鍵開關。概念示意：

```javascript
// 概念示意，非實際原始碼：安全題滿分 → 解鎖後續關卡
function gradeSafetyQuiz(answers, questions) {
  const correct = questions.filter((q, i) => answers[i] === q.correct).length;
  const isPerfect = correct === questions.length;   // 必須全對

  if (isPerfect) {
    unlockStage(tool, 'simulator');   // 解鎖操作模擬器
    saveProgress(tool, { safety: 'passed' });
  } else {
    showRetry(correct, questions.length);  // 顯示「X / N，再挑戰一次」
  }
  return isPerfect;
}
```

把「滿分」當門檻，而不是「六十分及格」，是因為**機具安全沒有及格邊緣**——一題答錯，背後可能就是一個受傷情境。要求學生把每一題都弄懂，正是這一關想傳達的態度。

## 用 localStorage 記住每件機具的闖關進度

學生不會一次把一件機具五關全部打完，更不會一次練完十幾件機具。所以每件機具的進度都用 **localStorage** 存在瀏覽器本機：闖到第幾關、安全題過了沒、認識了幾個部件，下次打開網頁自動接續。

因為**全站純前端、零後端**，沒有資料庫、沒有登入，進度天然就該存在使用者端。用 `localStorage` 的另一個好處是它離線可用、零延遲，學生在網路不穩的教室裡也不會卡。概念示意：

```javascript
// 概念示意，非實際原始碼：以工具為 key，存讀闖關進度
const KEY = 'ltt-progress';   // livingtech-tools progress

function saveProgress(tool, patch) {
  const all = JSON.parse(localStorage.getItem(KEY) || '{}');
  all[tool] = { ...(all[tool] || {}), ...patch };
  localStorage.setItem(KEY, JSON.stringify(all));
}

function loadProgress(tool) {
  const all = JSON.parse(localStorage.getItem(KEY) || '{}');
  return all[tool] || { parts: [], safety: 'locked', stage: 1 };
}
```

工具櫃頁面再讀回這份進度，就能在每張機具卡片上標出「已通過安全」「已完成」之類的狀態，學生一眼看得到自己練到哪。

## 操作模擬器：參數化的視覺模擬

解鎖之後的第四關「操作模擬器」，是讓學生**真的去試做一次**——當然是在螢幕上。這一關用**參數化的視覺模擬**：學生選定條件，系統依條件給出對應的視覺結果與提示。

以鑽孔模擬為例，學生可以選擇**多種材料 ×　多種鑽頭**的組合，模擬器再依組合給不同的視覺呈現與操作提示（例如某些材料配某種鑽頭要放慢、要不要先打中心點）。線鋸機則做了切割模擬，讓學生體會進料速度、轉彎半徑這類手感概念。概念示意：

```javascript
// 概念示意，非實際原始碼：依「材料 × 鑽頭」給出模擬回饋
function simulateDrill({ material, bit }) {
  const profile = drillTable[material]?.[bit];
  if (!profile) return hint("這個組合不建議，換一種鑽頭試試。");

  renderHole(profile.visual);        // 依組合畫出對應的孔／毛邊／焦痕
  return hint(profile.tip);          // 例如「先打中心點，避免鑽頭走位」
}
```

重點不在於模擬得多擬真，而在於**讓學生在零風險的環境裡，把「選錯組合會怎樣」先體驗過一輪**。等真的站到鑽床前，他腦中已經有一張「材料配鑽頭」的對照表。

## 為什麼整套做成純前端

技術選型上，全站刻意做成**純前端、零後端、部署在 GitHub Pages**。對教學現場來說，這個決定有幾個實際好處：

- **零維運成本**：沒有伺服器要顧、沒有資料庫要備份，學期中不會半夜掛掉。
- **隨開即用**：學生直接點網址就能用，不必註冊、不必登入，平板手機桌機都行。
- **進度存本機**：用 `localStorage` 存闖關進度，天然符合「無後端」架構，也避免蒐集學生個資的隱私顧慮。
- **好分享、好嵌入**：一個靜態網址，貼到 Google Classroom 或課程平台就能用。

代價是「跨裝置同步」做不到（換一台電腦進度不會跟著走），但對「課前先練熟」這個使用情境來說，這個取捨完全可以接受。

## 課堂怎麼用（對應 108 課綱）

- **機具安全操作（安全前置）**：實作課前先指派對應機具，要求學生**通過安全闖關**才算完成課前任務。學生帶著已經建立的安全意識進實作教室，老師的現場叮嚀就從「從零教起」變成「重點提醒」，大幅降低實作風險。
- **設計與製作**：操作模擬器讓學生在動手前先理解「材料 × 工具」的搭配邏輯，做專題時選料、選工具更有依據。
- **數位加工 / 工具認識**：三種視角的部件圖解，把「認識一件機具」這件事做得具體又有進度感，比看課本插圖更容易記住。
- **差異化教學**：手快的學生可以自己往後解鎖、多練幾件機具；手慢的反覆練同一件，**完全不佔用實體機台**。快慢差異第一次有了安放的地方。
- **討論素材**：第五關「故障 / 事故圖鑑」很適合拿來開放討論——「為什麼會壞？」「怎麼預防？」把安全教育從背誦變成思辨。

## 線上體驗

整套教具都在線上、純靜態、隨開即用。手邊有電腦或平板就能逛工具櫃、點開任何一件機具，從「認識部件」一路闖到「故障圖鑑」。歡迎同行老師直接拿去課堂用，或挑幾件當作實作前的課前任務。
