我丟了一句話給 AI：「把我近兩個月做的線上專案，**你自己點開、跑起來、錄畫面，寫好腳本讓我確認，再剪成一支介紹影片**。」

結果它真的做到了——而且後來連這篇文章、那頁完整製作紀錄，都是它自己生成的。

## 緣起：能不能讓 AI 自己把專案拍成影片？

這兩個月累積了九個線上專案——數位教具、工程設計平台、無影燈光學模擬、工程地景遊戲、水文即時監測、即時監控入口、握筆姿勢辨識、AI 操作 CAD⋯散在各個網址。要一個一個錄、配旁白、剪成片，是會吃掉一整個週末的工。

所以我把整件事**整個丟給 AI**，只保留一個關卡：腳本寫好要先給我確認。其餘自動。下面是它實際走的六步。

## 整條製作流程

![製作流程圖](/assets/blog/diagram-ai-video.svg)

### 1. 盤點：先讓 AI 自己找出「近期有在動」的專案

AI 沒有問我有哪些專案，而是自己用 `git log` 掃過本機各 repo 近兩個月的 commit 數，再用網路抓取每個線上網址的內容摘要，盤出「真的有實質開發、且能跑起來」的九個專案，順手把硬體類（沒辦法純螢幕錄製的）排除。**先盤點再動工**，這步決定了整支片的範圍。

### 2. 真實錄製：不是截圖，是讓網站「真的跑起來」

這是整個專案技術上最關鍵的一步。

一開始它試了「螢幕錄影」，但實機畫面很亂：瀏覽器分頁、擴充套件側欄、桌布全都入鏡。於是它換了一條更乾淨的路——**自己寫了一支無頭瀏覽器引擎**，用 Node 內建的 WebSocket 直接驅動 Chrome 的 DevTools Protocol（CDP），**自己控制每一格何時截圖**：

```js
// 概念：載入頁面後，自己一格一格推進動畫並截圖
await send('Page.navigate', { url });
for (const y of scrollPlan) {
  await evaljs(`window.scrollTo(0, ${y})`);   // 控制捲動
  await capture();                            // 想截才截
}
```

這樣做的好處是**截到的是真實互動，不是靜態截圖**：拉動無影燈的滑桿看陰影怎麼被稀釋、玩工程地景的猜謎遊戲、點切即時監控的各面板⋯都是網站「真的在跑」的畫面。它也順帶解掉了一個常見問題——即時更新的儀表板用傳統 `--screenshot` 會一直等不到「網路 idle」而卡死，自控截圖時機就沒這困擾。

### 3. 腳本：三幕結構＋分鏡審稿

有了素材，AI 把九個專案編成三幕——**把課本變成教具 / 連上真實世界的資料 / AI 與自造的前沿實驗**——個人作品站作頭尾。然後它做了一頁「膠卷分鏡審稿頁」，把每句旁白配上對應畫面截圖讓我逐句確認。這個**人類確認關卡**很重要：用詞、數字、節奏在這裡一次校準，後面才不用重剪。

### 4. 配音：找一個自然的台灣口語聲音

第一版用 macOS 內建語音「美佳」，但偏新聞主播腔。想換更日常的聲音時才發現：macOS `say` 列出的其他中文語音其實**沒安裝**，只會吐出 0.67 秒的空檔。於是改用 **edge-tts**（微軟線上神經語音），挑了台灣男聲「雲哲」(`zh-TW-YunJheNeural`)，第一人稱唸起來自然多了。

### 5. 剪輯：沒有字幕濾鏡，就用 PIL 逐格疊

這台機器的 ffmpeg 沒有 `drawtext`、也沒有 `libass`，**沒辦法直接燒中文字幕**。解法是用 Python 的 Pillow 把字幕、下標卡、幕標**逐格預渲成一條透明序列**，再用單一 overlay 疊回畫面。旁白比畫面長的段落，就從高解析畫格做 Ken-Burns 運鏡補滿。水文儀表板的中央地圖在無頭環境載不出來（一片白），就改裁成「左右資料欄並排」呈現。

### 6. 配樂、組裝與直式短影音

背景音樂用 numpy 合成一段柔和的襯底，最後串接十段、混音輸出。之後還延伸做了一支**直式 9:16、47 秒的短影音**（IG Reels／YT Shorts 共用）：模糊滿版背景＋清晰主畫面＋大字幕，濃縮九專案亮點。

## 踩雷紀錄：五個一度卡住的雷

失敗也是學習內容。這支片中途踩過、又一一解掉的雷：

| 症狀 | 原因 | 解法 |
|---|---|---|
| 字幕、下標**全程不顯示** | PIL 單格圖片只有 t=0 一格，fade 直接把整張 alpha 設成 0 | 改成**逐格預渲**透明序列，每格算好淡入淡出 |
| 某段 ffmpeg **失控、檔案爆到 422MB** | 把無限長的 `color=` 背景源當成 overlay 主軌，輸出永不結束 | overlay 加 `shortest=1`、補輸出 `-t` 邊界 |
| 開場標題片**長度爆成 324 秒** | `zoompan` 的 `d` 對 loop 影像設成總幅數，產生 N×N 幅 | 改 `d=1`、加輸入 framerate 與輸出 `-t` |
| 水文監測**中央一片白** | Leaflet 地圖瓦片在無頭環境載不出 | 量測欄位邊界，改取左右資料欄並排、深底置中 |
| 想換語音卻**只有空檔** | macOS `say` 列的語音其實沒安裝 | 改用 edge-tts 線上神經語音，先試聽再定 |

## 版本演進：從 v1 到 v2

- **v1**：美佳配音、4:17、個人站慢捲開場、片尾「歡迎拿去用」。
- **v2**（依我的回饋改版）：換**雲哲**口語男聲、總長 3:42；用詞**台灣化**（例如把「硬核」改成「進階」）；新增**開場標題卡＋八專案快剪 montage**；片尾改成邀請大家把建議留在留言區，並收在一句梗——「**就連這部影片，從腳本、配音到剪輯，全都是 AI 自己完成的唷**」。

## 這在課堂上能教什麼

對生活科技與資訊課，我覺得這個實驗有三個可談的點：

1. **把 AI 當「會動手的協作者」**：它不只回答問題，而是寫程式、跑瀏覽器、操作 ffmpeg——完成一條真實的工作流。這正是工程設計裡「**分解問題、選工具、迭代修正**」的縮影。
2. **失敗路徑的價值**：上面那五個雷，比「一次成功」更值得當教材。每個雷都是一次「假設 → 失敗 → 改用別的方法 → 學到」的完整循環。
3. **媒體製作的流程素養**：盤點素材、寫腳本、配音、剪輯、配樂——這支片把影像製作的完整流程攤開來，學生看得到「一支介紹影片背後有哪些工序」。
